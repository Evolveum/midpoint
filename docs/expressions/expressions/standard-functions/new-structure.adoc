= MidPoint standard functions
:page-toc: top
:toclevels: 1
:page-nav-title: Standard functions

The `MidpointFunctions` interface defines the built‑in midPoint functions that you can use in scripts.
These functions expose the core model operations (object creation, retrieval, modification, search, workflow, cryptography, etc.) as a concise, script‑friendly API.
All functions are public members of the interface;
they can be invoked directly from an expression code (e.g., Groovy, JavaScript, etc.) without importing any additional classes.

*The functions are available in link:https://github.com/Evolveum/midpoint/blob/master/model/model-api/src/main/java/com/evolveum/midpoint/model/api/expr/MidpointFunctions.java[MidpointFunctions.java]*. +
The link leads to the `master` branch.
Select the branch that corresponds to your midPoint version, e.g., `support-4.10`.

[NOTE]
====
*Notes on generic placeholders and overloads*

* Generic type parameters (`<T>` and `<F>`)
    ** `<T extends ObjectType>` is used for methods that operate on any concrete midPoint object type (e.g., `UserType`, `RoleType`, `OrgType`).
        The caller supplies the concrete class via a `Class<T>` argument, and the method returns a value of that same type, preserving compile‑time type safety.
    ** `<F extends FocusType>` is a specialization of `<T>` for focus objects (objects that can have assignments, projections, and a lifecycle), such as users, org units, and services.
        Methods that manipulate focus‑specific data (e.g., `recompute`, `isDirectlyAssigned`) use `<F>` to signal that the operation is only applicable to focus types.
    ** The placeholder name (`T` or `F`) is irrelevant to the runtime;
        it merely conveys the intended category of objects and enables the compiler to enforce correct usage.
* Var‑args (`...`) versus collection overloads
    ** Methods that accept a var‑args parameter (e.g., `executeChanges(ObjectDelta<? extends ObjectType>... deltas)`) allow callers to pass any number of arguments directly, without constructing a collection first.
        Internally the compiler creates an array, so the call is convenient for a small, known number of deltas.
    ** The corresponding collection overload (`executeChanges(Collection<ObjectDelta<? extends ObjectType>> deltas)`) is intended for scenarios where the deltas are already assembled in a `List`, `Set`, or another `Collection`. It avoids the extra array allocation that the var‑args version would incur and is more appropriate for bulk or dynamically built delta sets.
    ** Both overloads perform the same operation; the choice is purely a matter of API ergonomics and performance considerations. The var‑args version is annotated with `@SuppressWarnings("unchecked")` because Java cannot create a generic array at runtime, while the collection version does not have that limitation.

Below is the list of all standard midPoint functions, grouped thematically.

* parameters:

* returns:

== Search objects

=== Simple search

Searches for objects of the given type using the supplied query. Use `options` to influence retrieval and processing of the objects. This method has a very limited scaling capability as all the results are stored in memory. DO NOT USE on large datasets. Recommended usage is only when using queries that cannot return a large number of results (e.g., queries for unique values) or when combined with paging capability. For other cases use `searchObjectsIterative()`. The method also accepts a `TypedQuery<T>` as the query argument.

*`searchObjects()`*

* parameters:
** `Class<T> type, ObjectQuery query, Collection<SelectorOptions<GetOperationOptions>> options`
** `Class<T> type, ObjectQuery query`
** `TypedQuery<T> query`
** `TypedQuery<T> query, Collection<SelectorOptions<GetOperationOptions>> options`
* returns:
** `List<T>`
* throws:
** `CommunicationException`
** `ConfigurationException`
** `ExpressionEvaluationException`
** `ObjectNotFoundException`
** `SchemaException`
** `SecurityViolationException`

== Execute changes

=== Execute changes

Executes the provided object deltas. The order of execution is not defined and the implementation is free to determine the correct or most suitable ordering. All deltas must relate to analogous objects (e.g., user and linked accounts). The implementation may throw an error if the objects are not analogous. The implementation also implicitly links the objects if such a link is part of the data model. Only one delta per object is accepted. The OID provided in ADD deltas should be empty under usual circumstances. Use `options` to influence processing of the deltas.

*`executeChanges()`*

* parameters:
** `Collection<ObjectDelta<? extends ObjectType>> deltas, ModelExecuteOptions options`
** `Collection<ObjectDelta<? extends ObjectType>> deltas`
** `ObjectDelta<? extends ObjectType>... deltas`
* returns:
** `void`

** `ConfigurationException`

== Evaluation mode

=== Check evaluation mode

A stateful flag that tells a script whether it is currently handling a new value — one that will be written to the target object (e.g., values from the add/replace sections of an `ObjectDelta` or recomputed values slated for persistence) — or an old value that is being removed or read from the existing object (e.g., values from the delete section). Returns `true` for new, `false` for old, and `null` when the framework cannot determine the distinction or the context does not apply.

*`isEvaluateNew()`*

* parameters:
** The function does not accept any parameters.
* returns:
** `Boolean`
* throws:
** The method does not throw any exceptions.
* example:
** `if (midpoint.isEvaluateNew()) { /* prepare new value */ } else { /* handle old value */ }`

=== Check evaluation mode (Groovy-friendly)

A convenience method that allows writing `midpoint.evaluateNew` in Groovy. Functionally equivalent to `isEvaluateNew()`. Introduced to allow idiomatic Groovy property access after the Groovy upgrade in 4.8.

*`getEvaluateNew()`*

* parameters:
** The function does not accept any parameters.
* returns:
** `Boolean`
* throws:
** The method does not throw any exceptions.
